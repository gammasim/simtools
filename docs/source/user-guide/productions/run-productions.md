# Run Productions

The basic execution unit of a production is one simulation run. It is executed with
[simtools-simulate-prod](../applications/simtools-simulate-prod). Large productions are built
from many such runs, usually generated from the ECSV production grid described in
[Configure Productions](configure-productions.md).

## Run a Single Production Job

`simtools-simulate-prod` prepares the simulation configuration, writes run scripts, executes the
selected simulation software, validates the produced files, and reports the result.

The selected `simulation_software` controls which part of the chain is executed:

- `corsika`: run shower simulations only
- `sim_telarray`: run detector simulation using an existing CORSIKA input file
- `corsika_sim_telarray`: run the CORSIKA to sim_telarray chain using multipipe execution

Typical outputs are:

- CORSIKA, sim_telarray, and simtools log files
- sim_telarray eventio files and histogram files
- reduced event-data HDF5 files when `--save_reduced_event_lists` is used
- output and log file lists when `--save_file_lists` is used
- registration tarballs when `--pack_for_grid_register` is configured

Example command:

```bash
simtools-simulate-prod \
    --model_version 7.0.0 \
    --site North \
    --array_layout_name LSTN-01 \
    --primary gamma \
    --azimuth_angle North \
    --zenith_angle 20 deg \
    --energy_range 30 GeV 300 GeV \
    --core_scatter 10 500 m \
    --view_cone 0 deg 10 deg \
    --showers_per_run 5 \
    --run_number 1 \
    --save_reduced_event_lists
```

Example integration configurations are available in `tests/integration_tests/config`, including
`simulate_prod_gamma_40_deg_south_corsika_only.yml`,
`simulate_prod_gamma_40_deg_south_sim_telarray_only.yml`,
`simulate_prod_gamma_62_deg_south_check_output.yml`, and
`simulate_prod_proton_20_deg_north_check_output.yml`.

## Running simtools on HTCondor using Apptainers

Large productions are normally submitted to HTCondor using an Apptainer image. The submission
files are generated by
[simtools-simulate-prod-htcondor-generator][htcondor-generator].
The generator takes the ECSV job grid written by
[simtools-production-generate-grid](../applications/simtools-production-generate-grid) as input.

Apptainer images can be pulled directly from the GitHub package registry. Use the image tag
specified by the production configuration or production release notes:

```bash
apptainer pull --force \
    docker://ghcr.io/gammasim/simtools-prod:v0.27.1-v78010-v2025-11-30-rc-avx2
```

The generator supports either one image for all jobs or a mapping from image labels to image
paths. Label-to-image mappings are useful when one job grid is split across several production
images.

Example configuration from
`tests/integration_tests/config/simulate_prod_htcondor_generator_grid_horizontal.yml`:

```yaml
application: simtools-simulate-prod-htcondor-generator
configuration:
  apptainer_image:
    0.30.0: tests/resources/dummy_apptainer_image.sif
    0.29.0: tests/resources/dummy_apptainer_image.sif
  job_grid_file: tests/resources/production_job_grid_horizontal.ecsv
  output_path: htcondor_submit
  priority: 5
  simulation_output: ./simulation_output_path
```

Generate the submission files:

```bash
simtools-simulate-prod-htcondor-generator \
    --config tests/integration_tests/config/simulate_prod_htcondor_generator_grid_horizontal.yml
```

The generator writes:

- `simulate_prod.submit.sh`, the executable wrapper script
- one `simulate_prod.submit.<label>.condor` file per image label, or
  `simulate_prod.submit.condor` for a single image
- one matching `simulate_prod.submit.<label>.params.txt` file per submit file
- HTCondor log, error, and output directories

Before submission, copy the required environment variables into `env.txt` in the submission
directory. The file should include database credentials and simulation software paths expected
inside the container. For production Apptainer images, `SIMTOOLS_SIM_TELARRAY_PATH` is usually
set to `/workdir/sim_telarray`.

Submit jobs from the generated output directory:

```bash
cd htcondor_submit
condor_submit simulate_prod.submit.condor
```

For label-to-image configurations, submit the label-specific files, for example:

```bash
condor_submit simulate_prod.submit.0.30.0.condor
```

Monitor the jobs with `condor_q` and inspect the generated log directories for errors. Simulation
data products are written below the directory configured with `simulation_output`.
