simtools-simulate-prod#

Generate simulation configuration and run simulations.

Multipipe scripts will be produced as part of this application. Allows to run array layout simulation including shower and detector simulations

The entire simulation chain, parts of it, or nothing is executed:

  • shower simulations with CORSIKA only

  • shower simulations with CORSIKA which are piped directly to sim_telarray using the sim_telarray multipipe mechanism.

Command line arguments#

options#

–help

show this help message and exit

–corsika_file

Path to the CORSIKA input file (only relevant for simulation software ‘sim_telarray’).

–grid_output_path

Directory for output files for registering on the grid.

–save_file_lists

Save lists of output and log files.

–save_reduced_event_lists

Save reduced event lists with event data on simulated and triggered events. Saved with the same name as the sim_telarray output file (different extension).

–corsika_seeds

Use fixed random seeds for CORSIKA for testing purposes.

–sequential

Enables single-core mode (as far as possible); otherwise, CORSIKA and sim_telarray run in parallel.

–job_grid_file

Path to an ECSV job grid file produced by simtools-production-generate-grid. When provided, the selected row defines production parameters and must not be combined with manual production arguments such as ‘–zenith_angle’.

–job_grid_row

1-based index of the row to read from the file given by ‘–job_grid_file’. Defaults to 1 (first row).

Default: 1.

simulation model#

–model_version

Simulation production model version

–overwrite_model_parameters

File name to overwrite model parameters from DB with provided values

–site

Observatory site (e.g., North, South)

–telescope

telescope model name (e.g., LSTN-01, SSTS-design, …)

–array_layout_name

Array layout name(s) (e.g., CTAO-North-Alpha, LSTN-01). Telescope names are assumed as single-telescope layouts.

–array_element_list

list of array elements (e.g., LSTN-01, LSTN-02, MSTN).

–ignore_missing_design_model

Ignore missing design model definition of DB

simulation software#

–simulation_software

Simulation software steps.

Default: corsika_sim_telarray. Choices: corsika, sim_telarray, corsika_sim_telarray.

simulation configuration#

–primary

Primary particle to simulate. (choices for common names: gamma, electron, positron, muon+, muon-, proton, neutron, helium, carbon, nitrogen, silicon, iron; use ‘–primary_id_type’ to use other particle ID types).

Required.

–primary_id_type

Primary particle ID type

Default: common_name. Choices: common_name, corsika7_id, pdg_id.

–azimuth_angle

Telescope pointing direction in azimuth. It can be in degrees between 0 and 360 or one of north, south, east or west. North is 0 degrees and the azimuth grows clockwise (East is 90 degrees).

–zenith_angle

Zenith angle in degrees (between 0 and 180).

Default: 20.0 deg.

–showers_per_run

Baseline number of CORSIKA showers per run.

–run_number_offset

Offset added to run number when executing a simulation.

–run_number

Run number to be simulated.

Default: 1.

–event_number_first_shower

Event number of first shower

Default: 1.

–correct_for_b_field_alignment

Correct for B-field alignment

Default: True.

–curved_atmosphere_min_zenith_angle

Minimum zenith angle (deg) for using curved-atmosphere CORSIKA binaries.

Default: 65.0 deg.

shower parameters#

–eslope

Slope of the energy spectrum.

Default: -2.0.

–energy_range

Energy range of the primary particle (min/max, e.g., ‘10 GeV 5 TeV’).

Default: (<Quantity 3. GeV>, <Quantity 330. TeV>).

–view_cone

View cone radius for primary arrival directions (min/max value, e.g. ‘0 deg 10 deg’).

Default: [‘0 deg 0 deg’].

–core_scatter

Scatter radius for shower cores (number of use; scatter radius).

Default: [‘10 10000 m’].

corsika configuration#

–corsika_he_interaction

High-energy interaction model for CORSIKA (default fallback: epos).

–corsika_le_interaction

Low-energy interaction model for CORSIKA (default fallback: urqmd).

sim_telarray configuration#

–sim_telarray_instrument_seed

Random seed used for sim_telarray instrument setup.

–sim_telarray_random_instrument_instances

Number of random instrument instances initialized in sim_telarray.

Default: 1.

–sim_telarray_seed

Random seed used for sim_telarray simulation. Single value: seed for event simulation. Two values: [instrument_seed, simulation_seed] (use for testing only).

database configuration#

–db_api_user

database user

–db_api_pw

database password

–db_api_port

database port

–db_server

database server address

–db_api_authentication_database

database with user info (optional)

–db_simulation_model

name of simulation model database

–db_simulation_model_version

version of simulation model database
